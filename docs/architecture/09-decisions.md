# 9 Architecture Decisions

[← Previous: 8 Cross-cutting Concepts](08-concepts/08-0-concepts.md) | [Table of Contents](README.md) | [Next: 10 Quality Requirements →](10-quality.md)

We use [Architecture Decision Records (ADRs)](https://cognitect.com/blog/2011/11/15/documenting-architecture-decisions) to capture the "why" behind significant architectural choices. Each ADR follows the Nygard structure (Context, Decision, Consequences) and lives in the repository alongside the code it governs.

## ADR Location and Process

ADRs are stored in [`docs/adr/`](../adr/) within this repository. See the [ADR README](../adr/README.md) for the full process, template, and naming conventions.

New ADRs should be proposed via pull request when making decisions that affect system structure, quality characteristics, external dependencies, or construction techniques.
