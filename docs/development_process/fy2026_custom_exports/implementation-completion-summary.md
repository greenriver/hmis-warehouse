## 🏗️ **Technical Architecture**

### **Data Flow**
1. **User Selection**: User selects custom files via UI (select2 multi-select)
2. **Form Submission**: `custom_file_types: ["CustomGender.csv", "CustomSexualOrientation.csv"]`
3. **Controller Processing**: Parameters accepted and passed to filter model
4. **Filter Validation**: Custom file types validated against available options
5. **Job Scheduling**: Export job receives custom_file_types parameter
6. **Export Generation**: Base exporter dynamically includes custom file classes
7. **File Generation**: Custom exporters delegate to standard exporters for data scope
8. **ZIP Creation**: Custom CSV files included alongside standard HMIS files

### **Dynamic Model Generation**
- **Command**: `HmisCsvTwentyTwentySix::CustomFileManager.bootstrap_custom_models!`
- **Generates**: Loader, Importer, and Exporter model files
- **Configuration**: Based on YAML files in `config/custom/` directory
- **Delegation**: Custom exporters delegate `export_scope` to standard exporters
- **Mapping**: Uses `augments_export_class` configuration for proper delegation

### **Version Compatibility**
- **FY2026**: Full custom file support with UI and export functionality
- **Prior Versions**: Graceful degradation (no UI shown, empty arrays handled)
- **Progressive Enhancement**: Feature doesn't break existing functionality
