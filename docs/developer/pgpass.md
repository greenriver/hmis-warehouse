# PostgreSQL Password File (.pgpass)

## Overview

The `.pgpass` file is a PostgreSQL authentication file that stores connection credentials for multiple databases. It allows command-line tools (like `psql`, `pg_dump`, etc.) to connect to databases without prompting for passwords.

## File Format

Each line represents a database connection with the following format:

```
hostname:port:database:username:password
```

- `hostname`: The database host (can be `*` for any host)
- `port`: PostgreSQL port (default is `5432`, can be `*` for any port)
- `database`: Database name (use `*` for any database)
- `username`: PostgreSQL user
- `password`: PostgreSQL password

## Security

The `.pgpass` file must have permissions `600` (read/write for owner only):

```bash
chmod 600 ~/.pgpass
```

This is enforced automatically by the generation scripts.

## Generation

The `.pgpass` file is generated from environment variables using `bin/generate_pgpass.rb`.

### Supported Databases

The script generates entries for the following databases:

- **Primary**: `DATABASE_HOST`, `DATABASE_PORT`, `DATABASE_USER`, `DATABASE_PASS`
- **Warehouse**: `WAREHOUSE_DATABASE_HOST`, `WAREHOUSE_DATABASE_PORT`, `WAREHOUSE_DATABASE_USER`, `WAREHOUSE_DATABASE_PASS`
- **Health**: `HEALTH_DATABASE_HOST`, `HEALTH_DATABASE_PORT`, `HEALTH_DATABASE_USER`, `HEALTH_DATABASE_PASS`
- **Reporting**: `REPORTING_DATABASE_HOST`, `REPORTING_DATABASE_PORT`, `REPORTING_DATABASE_USER`, `REPORTING_DATABASE_PASS`
- **CAS** (optional): `CAS_DATABASE_HOST`, `CAS_DATABASE_PORT`, `CAS_DATABASE_USER`, `CAS_DATABASE_PASS`

### Generation Locations

- **Development**: Generated via `bin/setup` after `.env.local` is created
- **Docker Development**: Generated by `docker/app/entrypoint.dev.sh` before starting the application
- **Docker Production**: Generated by `docker/app/entrypoint.sh` after secrets are sourced

### Manual Generation

To manually generate the `.pgpass` file:

```bash
bundle exec ./bin/generate_pgpass.rb > ~/.pgpass
chmod 600 ~/.pgpass
```

## Usage

With a properly configured `.pgpass` file, you can connect to databases without entering passwords:

```bash
psql -h your-host -U your-user database-name
```

This is useful for:
- Automated scripts and CI/CD pipelines
- Background jobs
- Docker containers
- Development workflows
